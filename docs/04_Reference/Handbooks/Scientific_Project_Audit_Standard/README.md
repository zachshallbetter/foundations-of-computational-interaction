# Scientific Project Audit and Grading Standard

Version: 0.1.0-draft
Status: Draft for internal validation

## Purpose

This standard defines a domain-independent methodology for auditing, reviewing, and grading scientific projects. It evaluates whether a project's claims are scientifically knowable, justified, reproducible, comparable, and proportionate to their complexity.

The standard is designed for conceptual papers, formal models, ontologies, measurement systems, empirical studies, computational implementations, datasets, replications, syntheses, and research programs.

## Governing principle

Scientific quality is not a single property. A project may be conceptually strong, empirically immature, statistically precise, ontologically incoherent, or operationally complete but scientifically invalid. The audit therefore uses:

1. fatal validity gates;
2. project-type-specific dimensions;
3. evidence-confidence adjustments;
4. dependency integrity analysis;
5. maturity classification;
6. comparative-gain and complexity analysis;
7. a bounded final grade and review verdict.

A weighted average may summarize eligible work, but it may not conceal a failed foundational dependency.

## Canonical audit sequence

```text
Scope and classification
-> Claim inventory
-> Fatal validity gates
-> Dependency audit
-> Dimension review
-> Evidence-confidence assessment
-> Formula and measurement audit
-> Comparative-gain and complexity review
-> Maturity classification
-> Grade and verdict
-> Remediation plan
```

## Relationship to existing handbooks

This standard does not replace the Measurement Handbook, Experimental Handbook, or Replication Handbook. It governs how those methods are selected, inspected, and graded. Where a project invokes a Measurement Contract, study protocol, replication protocol, or other canonical contract, the audit must evaluate conformance to that artifact.

## Normative language

The terms `must`, `must not`, `required`, and `prohibited` are normative. The terms `should`, `recommended`, and `may` are advisory unless another canonical artifact states otherwise.
