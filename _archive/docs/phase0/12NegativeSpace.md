# Deliverable 12: Negative Space

**Program:** Computational Interaction Science (CompInt)  
**Date:** 2026-07-17  
**Status:** Canonical Reference Model  

This document defines the **Negative Space** of Computational Interaction Science (CompInt). It explicitly delineates what the program does not study, what it does not claim, and the conditions under which it collapses into existing disciplines.

---

## 1. What is NOT Part of the Field

To maintain epistemic discipline and avoid theoretical bloat, CompInt excludes the following adjacent domains:

*   **UX Design Methods and Design Systems:** We do not develop design guidelines, aesthetics tips, or library components (e.g., CSS button styles). Prescribing how a button *should* look belongs to design practice, not interaction science.
*   **Usability Heuristics:** Traditional list-based evaluation heuristics (e.g., "provide feedback," "ensure consistency") are rules of thumb. CompInt focuses on formal, testable, and quantitative state-transition and opportunity invariants.
*   **Personality and Psychological Profiling:** Models claiming to categorize users into personality types (e.g., Myers-Briggs, Big Five) or cognitive archetypes are excluded. Traces are evidence of action history and opportunity, not of permanent psychological identity.
*   **Learning Styles:** The theory of distinct human "learning styles" (e.g., visual vs. auditory learners) is empirically unvalidated. CompInt studies prior exposure history vectors, not biological cognitive typologies.
*   **Analytics and Marketing Telemetry Platforms:** Standard web analytics (e.g., Google Analytics tracking pageviews and bounce rates) are designed for marketing funnel optimization. They do not record system state, preconditions, or opportunity alternatives, and are unsuitable as scientific evidence.
*   **Accessibility Standards:** Compliance checklists (e.g., WCAG) are regulatory instruments. CompInt studies accessibility from the perspective of formal projection mappings and capability-enablement matching.
*   **AI Interface Models:** Generative UI models (e.g., AI generating layout designs on the fly) are engineering tools. CompInt studies how these generated structures behave as projections, but it does not develop generation algorithms.
*   **Gamified Telemetry:** Designing games specifically to extract user data is a product design task, not a scientific methodology.

---

## 2. Epistemic Collapse Criteria
The research program will immediately halt or merge into adjacent disciplines if any of the following three conditions are met:

### 2.1 The HCI-CogSci Explanatory Sufficiency
If controlled experiments show that combining standard Human-Computer Interaction usability metrics and existing Cognitive Psychology models (e.g., ACT-R) explains and predicts user interface transfer and adaptation without any material loss in accuracy compared to the CompInt contract stack, then:
*   **Action:** The program collapses, and CompInt is abandoned.

### 2.2 The Telemetry Branding Collapse
If the Opportunity Profile, Strategy Contract, and trace schemas collapse in practice into simple, un-contextualized telemetry event counts (clicks/time) without reference-state tracking, then the program has failed to maintain its core separation of concerns.
*   **Action:** The program is merged into standard software analytics engineering.

### 2.3 The Metaphysical Inflation
If future concept papers drift from modeling system projections and start claiming to explain human culture, social organizations, or independent cognitive mechanisms without exposing operationalized exposure variables, the program violates its first principles.
*   **Action:** The papers are rejected, and the program is forced to return to a deterministic subset.
