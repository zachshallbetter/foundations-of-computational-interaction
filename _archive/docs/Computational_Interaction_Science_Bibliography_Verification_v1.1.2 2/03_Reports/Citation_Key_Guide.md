# Citation Key Guide

Canonical keys are preserved from the richest retained source record to avoid breaking existing citations. Duplicate keys are listed as aliases in the audit.

## Policy

1. Existing canonical keys are immutable within Version 1.x.
2. New keys use `familyYYYYshorttitle`; add a lowercase suffix only for collisions.
3. Do not encode journal, publisher, or edition in a key.
4. Key redirects must be recorded in the duplicate report.
5. A future major version may migrate keys only with an automated mapping file.
