# Reference Maintenance Guide

1. The master BibTeX file is the system of record.
2. Add a reference only after title, creator, year, and work type are verified.
3. Prefer publisher pages, Crossref, DataCite, library catalogs, standards bodies, and the work itself.
4. Normalize DOI as the bare identifier; store resolvable URL separately if needed.
5. Validate ISBN checksums and edition specificity.
6. Never infer issue or page ranges from neighboring records.
7. Merge duplicates only after work identity is confirmed.
8. Preserve aliases and retired citation keys in a migration table.
9. Regenerate CSL JSON and formatted references after each approved change.
10. Run completeness, duplicate, identifier-format, and link checks before release.
